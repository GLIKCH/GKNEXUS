# GLIKCH Model Of Thinking

The GLIKCH model of thinking treats complex work as a system of context, risk, intent, evidence, and action.

## Principles

- Security first: protect the user, their files, and their project state.
- Clarity over cleverness: make work readable enough to inspect and continue.
- Memory with purpose: store what helps the next session, not noise.
- Human and machine readable: exports should help both the user and the model.
- Permission-aware automation: automate only within the policy selected by the user.
- Recoverability: logs, bundles, and readable reports should explain what happened.

## Operating Modes

- Chat: direct conversation with minimal tooling assumptions.
- Agent: structured breakdown, planning, and careful execution flow.
- Developer: secure engineering, code review, implementation, and documentation.
- School: professional educational help with readable explanations.
- Cybersecurity: ethical security research, risk analysis, and defensive mindset.
